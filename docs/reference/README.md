# Reference-документы

В этой директории собраны документы, которые полезны не как первое чтение, а как точный справочный слой. Их задача — не вводить в проект, а давать детальные формулировки, полные перечни и исторически накопленный объём сведений.

Если вы только входите в проект, лучше начинать не отсюда, а с:

- `../../README.md`
- `../user/guide.md`
- `../dev/guide.md`

## Что здесь лежит

### `appendices.md`

Справочные приложения к основным документам:

- карта директорий проекта
- таблица портов и протоколов
- набор минимальных примеров команд и запросов

### `../dcp2.md`

Формальная спецификация бинарного протокола DCP2. Это отдельный нормативный документ, который имеет смысл читать как protocol reference, а не как вводный текст.

## Как этим пользоваться

Хорошая рабочая схема такая: сначала читать руководство по роли, затем открывать точный тематический документ по нужной подсистеме, и только потом пользоваться `appendices.md` как вспомогательным справочным слоем.
